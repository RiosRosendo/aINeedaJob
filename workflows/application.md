# Workflow: Application

## Objective

Submit a job application on behalf of the user using the tailored CV and cover letter generated by the CV Tailoring Agent. Use API integration when available, browser automation as fallback.

---

## Trigger

- Automatic: triggered by CV Tailoring Agent when Application status = `documents_ready`
- Manual: user can trigger from dashboard for any application with status `documents_ready`

---

## Required Inputs

| Input | Source | Description |
|---|---|---|
| `application_id` | Database | The application to submit |
| `job_id` | Database | The target job |
| `user_id` | Database | The user applying |
| `cv_version_url` | applications table | Tailored CV PDF path |
| `cover_letter_url` | applications table | Cover letter PDF path |
| `url` | jobs table | Original job posting URL |
| `source` | jobs table | Job board source (adzuna, themuse, etc.) |
| `title` | jobs table | Job title |
| `company` | jobs table | Company name |

---

## Tools Used

| Tool | Purpose |
|---|---|
| `tools/apply_via_api.py` | Submit application through job board API |
| `tools/apply_via_browser.py` | Submit application via Playwright browser automation |
| `tools/update_application.py` | Update Application status in database |
| `tools/create_notification.py` | Notify user of application result |

---

## Steps

### Step 1 — Load application data
- Query `applications` and `jobs` tables
- If not found: log error, abort
- If status is not `documents_ready`: log warning, abort
- Download CV and cover letter PDFs from cloud storage
- If files not accessible: log error, notify user, abort

### Step 2 — Select submission method
Choose method based on job source:

| Source | Method |
|---|---|
| Adzuna | API (if apply endpoint available) → fallback to browser |
| The Muse | API (if apply endpoint available) → fallback to browser |
| Direct company URL | Browser automation |
| Unknown | Browser automation |

### Step 3A — Submit via API
- Call `tools/apply_via_api.py` with job source, application data, CV, and cover letter
- On success (200/201): proceed to Step 4
- On 401/403: log "API auth failed", fallback to Step 3B
- On 429: wait 60 seconds, retry once. If fails again: fallback to Step 3B
- On 404: job posting no longer exists, mark application as `job_expired`, notify user, abort

### Step 3B — Submit via browser automation
- Call `tools/apply_via_browser.py` with the job URL
- Playwright opens the job posting in a headless browser
- Actions performed in order:
  1. Navigate to job URL
  2. Locate and click the apply button
  3. Fill in name, email from UserProfile
  4. Upload CV PDF
  5. Upload cover letter PDF if field exists
  6. Fill any additional required fields with UserProfile data
  7. Submit the form
- On success: proceed to Step 4
- On CAPTCHA detected: pause, mark as `requires_manual`, notify user to complete manually
- On form field not recognized: mark as `requires_manual`, notify user
- On timeout (30 seconds): retry once, then mark as `requires_manual`

### Step 4 — Confirm submission
- Verify confirmation page or confirmation email trigger
- Update `Application` status to `applied`
- Record `applied_at` timestamp

### Step 5 — Notify user
- Call `tools/create_notification.py` with:
  - `type`: "application_submitted"
  - `message`: "Applied to {job_title} at {job_company}"
  - `job_id`

### Step 6 — Log result
- Write a record to `agent_logs`:
  - `user_id`
  - `job_id`
  - `application_id`
  - `agent`: "application"
  - `method`: "api" or "browser"
  - `status`: "applied" / "requires_manual" / "job_expired" / "failed"
  - `timestamp`

---

## Expected Output

- Application status updated to `applied` or `requires_manual`
- User notified of result
- Email Monitoring Agent begins watching for responses

---

## Error Handling

| Situation | Action |
|---|---|
| Job posting expired (404) | Mark as `job_expired`, notify user |
| CAPTCHA encountered | Mark as `requires_manual`, notify user with URL |
| Unknown form fields | Mark as `requires_manual`, notify user with URL |
| Both API and browser fail | Mark as `failed`, notify user, log error |
| CV file not accessible | Abort, notify user to re-upload |
| Confirmation not detected | Mark as `applied_unconfirmed`, log warning |

---

## Notes

- Browser automation is V2 priority. In V1, focus on sources with API support (Adzuna, The Muse).
- Never submit an application without a valid CV attached. If the CV file is missing, abort.
- The `requires_manual` status means the user must complete the application themselves. Always include the direct job URL in the notification so they can do it in one click.
- LinkedIn and Indeed block automated submissions aggressively. Do not attempt browser automation on these platforms without proper anti-detection measures in place.
- Every application submitted must be logged regardless of outcome. The user has a right to know what was submitted on their behalf.
